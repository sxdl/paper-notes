系统设计：基于智能体的“记事簿”架构

1. 用户自定义存储信息分类

  • 用户可以通过界面或简单指令，自定义信息的分类和子分类。
  • 示例：用户定义以下分类：
    • 任务：工作任务、学习任务
    • 计划：短期计划、长期计划
    • 灵感：随笔、待研究
    • 其他：自定义分类

2. 多智能体分工

每个智能体专注于一个特定任务，彼此协同完成整体工作。

智能体名称 职责 输入输出格式 Prompt 关键点
分类判断智能体 判断用户输入是否符合某分类，并分发到对应子任务 用户输入文本 -> 分类标签 了解用户分类定义及语义规则
存储智能体 将信息存储为用户定义的结构化格式 分类标签 + 输入文本 -> JSON结构 学习分类规则与结构化模板
查询智能体 根据用户查询请求，筛选并返回相关信息 查询指令 -> 筛选结果 理解上下文，提取限定词及范围条件
规划智能体 基于存储信息，为用户生成任务或时间规划 存储信息 + 指令 -> 规划建议 考虑优先级、时间冲突等约束条件

智能体职责详解

1. 分类判断智能体

  • 职责：根据用户输入，判断其属于哪个分类，或是否需要新增分类。
  • 示例：
    • 用户输入：“下周二要完成项目报告。”
    • 智能体输出： 复制代码 分类：任务 标签：工作任务

2. 存储智能体

  • 职责：将信息按分类规则转换为结构化存储格式。
  • 示例：
    • 输入：分类：任务 | 标签：工作任务 | 内容：下周二要完成项目报告
    • 输出（JSON）： json 复制代码 { "分类": "任务", "标签": "工作任务", "内容": "完成项目报告", "时间": "2025-01-02", "优先级": "高" }

3. 查询智能体

  • 职责：解析用户查询意图，匹配相关信息并返回结果。
  • 示例：
    • 用户输入：“有哪些待完成的工作任务？”
    • 智能体输出： diff 复制代码 查询条件：分类=任务 & 标签=工作任务 & 状态=未完成 查询结果： - 完成项目报告（截止日期：2025-01-02）

4. 规划智能体

  • 职责：基于存储信息，生成时间规划或任务清单。
  • 示例：
    • 用户输入：“帮我规划明天的任务。”
    • 智能体输出： markdown 复制代码 明日计划： 1. 上午：完成PPT（高优先级）。 2. 中午：开会（固定时间）。 3. 下午：买书（灵活安排）。

多智能体协同工作流程

  1. 分类与存储：
    • 用户输入信息，分类判断智能体识别分类，存储智能体负责结构化存储。
  2. 任务查询：
    • 用户查询时，查询智能体解析意图，筛选出匹配内容。
  3. 任务规划：
    • 用户需要规划时，规划智能体结合存储信息和规则生成计划。
  4. 新增分类动态扩展：
    • 用户定义新分类，分类判断智能体自动更新规则，支持实时扩展。

技术实现

1. Prompt设计

  • 分类判断： php 复制代码 这是用户定义的分类规则：<规则详情>。 现在根据规则判断以下文本属于哪个分类，并返回对应分类标签。 输入：<用户输入> 输出：分类：<分类名称> | 标签：<子分类>
  • 存储格式化： php 复制代码 以下是分类信息与内容： 分类：<分类名称> 标签：<子分类> 内容：<用户输入内容> 请将上述信息存储为结构化JSON格式，并标注时间、优先级等关键信息。
  • 查询处理： php 复制代码 用户输入了一条查询请求：<查询内容> 请从已存储信息中，提取满足条件的内容，返回结果。
  • 规划生成： php 复制代码 用户希望根据存储信息规划任务：<指令详情> 请根据任务优先级、时间安排生成合理的规划建议。

2. 数据存储技术

  • 数据库：使用SQLite、MongoDB等轻量化数据库，存储结构化数据。
  • 缓存系统：基于Redis加速查询响应。

总结

“记事簿”的多智能体架构让系统更灵活，可扩展性更强，每个智能体分工明确，能有效处理不同任务。用户可自定义分类和存储规则，享受高度个性化的智能助手体验。这种设计既贴近实际需求，又体现了AI的智能和人性化特点。
